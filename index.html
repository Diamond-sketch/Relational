Summary

Slide 25Disk Format

Summary
  • Each disk has two surfaces where data is stored.
  • If a disk stores data on only one of its surface, it is called single-sided.
  • If a disk stores data on both surfaces, it is called double-sided.
  • Each surface is divided into multiple concentric circles of small width, called tracks.
  • Tracks are separated by inter-track gaps.
  • Each track is typically divided into multiple sectors, so-called disk blocks.
  • Sectors are separated by inter-sector gaps, so-called inter-block gaps.
  • Gaps are used to help identifying the beginning of sectors........…...….............
Answer / solution

Disk format: A disk has two surfaces. Single-sided uses one surface; double-sided uses both. Each surface is divided into tracks; tracks are separated by inter-track gaps. Each track is divided into sectors/disk blocks; sectors are separated by inter-sector/inter-block gaps. Gaps identify sector boundaries.

Slide 36Disk Access Time

Summary
  • Seek time – time for the disk to position the head assembly at the proper cylinder.
  • Latency time (i.e., rotational delay) – time for the disk to rotate the beginning of the desired sector containing the block into position under the read/write head.
  • Transfer time – time for the disk to transfer blocks of data between disk and main memory.
  • i.e., transfer sectors of the block and any gaps between them.
  • Normally, much less than seek and latency time.
  • Bulk transfer — several consecutive sectors on the same track or cylinder are transferred.
  • Disk access time in order of milliseconds is quite high compared with CPU processing time...
Answer / solution

Disk access time: Disk access time = seek time + rotational latency/delay + transfer time. Seek positions the head at the right cylinder. Latency waits for the sector to rotate under the head. Transfer time moves block data between disk and memory.

Buffering: Buffering = pre-loading data into memory independently/in parallel with CPU processing while a process requests I/O. Buffer = memory pages reserved to temporarily store disk blocks.

Slide 44File Allocation Techniques

Summary
  • File allocation techniques include contiguous allocation, linked allocation, cluster allocation, and indexed allocation.
  • Contiguous allocation keeps file blocks in consecutive disk blocks; it is fast for reading the whole file but slow when expanding the file.
  • Linked allocation stores a pointer to the next file block in each file block; it is easy to expand but slow to read the whole file.
  • Cluster allocation connects linked clusters of contiguous blocks.
  • Indexed allocation creates an index file with blocks that contain pointers to the data file blocks.
Answer / solution

File allocation: Contiguous allocation, linked allocation, cluster allocation, and index allocation.

Slide 109Shadow Paging

Summary
  • Advantages.
  • No overhead of maintaining log file.
